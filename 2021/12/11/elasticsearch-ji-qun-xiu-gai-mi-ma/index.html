ElasticSearch集群修改密码

1. 停止写入es的相关程序

这里基本上是Logstash，需要将Logstash停机。如果logstash有多个，都需要停机。


$ sudo systemctl stop logstash

2. 修改es密码

2.1 创建新的管理员用户

$ sudo /opt/es/bin/elasticsearch-users useradd changepasswd -r superuser

// 设置密码为：#EDCxsw2

2.2 curl请求，利用新管理员用户设置es密码：

curl -XPUT -u changepasswd:#EDCxsw2 http://10.3.61.242:9200/_xpack/security/user/elastic/_password -H "Content-Type: application/json" -d '{"password":"!QAZxsw2#EDc"}'

开始请求验证：

解决方式如下：


$ curl -XDELETE -u 'changepasswd:#EDCxsw2' http://10.3.61.242:9200/.security-*

// 删除之后重启es，如果是集群，则需要重启所有节点的es
$ sudo systemctl daemon-reload && sudo systemctl restart es

删掉所有的认证信息，现在需要重新设置密码，操作如下：


$ sudo /opt/es/bin/elasticsearch-setup-passwords interactive

es集群恢复后，删除此前添加的用户：

$ sudo /opt/es/bin/elasticsearch-users userdel changepasswd

2.4 修改kibana连接密码

$ sudo vim /opt/kibana/config/kibana.yml

server.port: 55561
server.host: "0.0.0.0"
elasticsearch.hosts: ["http://127.0.0.1:9200"]
#elasticsearch.hosts: ["http://127.0.0.1:55520"]
elasticsearch.username: "elastic"
# 修改下面的密码信息为最新
elasticsearch.password: "!QAZxsw2#EDc"
i18n.locale: zh-CN

// :wq 保存退出

// 重启kibana
$ sudo systemctl daemon-reload
$ sudo systemctl restart kibana

通过浏览器进行访问验证，这时候kibana访问可以读取之前的日志信息，但是最近几分钟或者10多分钟因为logstash停机，是没有日志的。

2.5 修改logstash中的密码信息并重启


$ sudo vim /opt/logstash-conf/leslog.conf

# Sample Logstash configuration for creating a simple
# Beats -> Logstash -> Elasticsearch pipeline.

input {
  kafka {
    bootstrap_servers => ["10.3.61.246:9092"]
    group_id => "test-consumer-group"
    topics => ["leslog"]
    auto_offset_reset => "earliest"
    consumer_threads => "1"
    client_id => "leslog"
    enable_auto_commit => "true"
    decorate_events => "true"
    auto_commit_interval_ms => "100"
    #codec => "json"
    codec => json {
        charset => "UTF-8"
    }
    type => "kafka-to-elas"
  }
}

output {
  #if [type] == "kafka-to-elas" {
    elasticsearch {
      hosts => ["10.3.61.242:9200"]
      index => "leslog-%{+YYYY.MM.dd}"
      user => "elastic"
      # 只修改这个位置的密码信息！
      password => "!QAZxsw2#EDc"
      action => "update"
      document_id => "%{id}"
      doc_as_upsert => true
      script => 'ctx._source.content = ctx._source.content + ";" + params.event.get("content");ctx._source.type = params.event.get("type")'
      script_type => "inline"
    }
  #}
}

// :wq 保存退出

$ sudo systemctl daemon-reload && sudo systemctl restart logstash

logstash启动后，会逐步从kafka中读取挤压的日志信息，大概10分钟左右就可以看到之前时间段之内缺失的日志信息了。

3. 总结

如果忘记密码，需要使用通过添加管理员，删除安全索引的方式进行！
