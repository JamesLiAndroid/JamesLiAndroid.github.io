xxl-job 在k8s中ip地址更换导致定时任务无法调用的问题

xxl-job在k8s中的部署方式

xxl-job服务端的部署方式

直接使用官方镜像进行部署，部署的yaml文件如下：

apiVersion: apps/v1
kind: Deployment
metadata:
  namespace: test-les
  name: xxl-job-admin
spec:
  replicas: 1
  selector:
    matchLabels:
      app: xxl-job-admin
  template:
    metadata:
      labels:
        app: xxl-job-admin
    spec:
      nodeSelector:
        xxljob: "on"
      containers:
      - name: xxl-job-admin
        image: xuxueli/xxl-job-admin:2.3.0
        imagePullPolicy: Always     # 优先使用本地镜像
        ports:
        - containerPort: 8080
        env:
        - name: PARAMS   # 定义变量，用来接收sql的用户/密码 mysql为k8s集群内的service名称，在k8s集群内部可以直接使用service名称，因为集群默认做了coredns解析
          value: "--spring.datasource.url=jdbc:mysql://192.168.166.115:3306/xxl_job?Unicode=true&characterEncoding=UTF-8&useSSL=false --spring.datasource.username=root --spring.datasource.password=123456"
---
apiVersion: v1
kind: Service
metadata:
  namespace: test-les
  name: xxl-job-admin
  labels:
    app: xxl-job-admin
spec:
  type: NodePort
  ports:
  - name: http
    port: 8080
    targetPort: 8080
    nodePort: 30180
    #protocol: TCP
    #name: http
  selector:
    app: xxl-job-admin

让该配置文件生效：


$ kubectl apply -f xxl-job-admin.yaml

这样xxl-job的server端就已经部署完毕，查看pod、deploy和svc资源情况：

访问地址192.168.166.111:30180/xxl-job-admin/即可打开前端页面。

xxl-job客户端配置以及单个任务编写

在微服务侧，添加xxl-job的配置，这里我们做了封装，因此展示如下：

    admin: 
      addresses: http://192.168.166.111:30180/xxl-job-admin

      # addresses: http://xxl-job-admin.test-les:8080/xxl-job-admin

问题现象

在服务调用时出现了间歇性调用失败的情况，我们用一个10秒执行一次的空定时任务进行测试，发现大概每隔20分钟就会出现一波失败的任务。

成功任务的截图如下：

失败任务的截图如下：

二者对比，在成功的任务中，显示的调度机器ip为正常xxl-job-admin pod所在的IP地址，也是k8s中的ip地址，但是在失败的任务重，显示的调度机器ip地址为172.17.0.2。调度机器ip发生了变化。

上述问题是自动执行时发生，在手动执行时，没有错误信息发生！

查看网卡得知，172.17.0.2来自于docker0网卡，应该是docker容器对外访问的统一ip地址了。

尝试过的解决办法

  1. 在微服务侧，修改获取ip地址的方法，以及屏蔽其余网卡，无效。

例如在pom.xml中引入common包：


<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-commons</artifactId>
    <version>3.1.0</version>
</dependency>

在配置文件中添加：


# 设置ip地址段
spring.cloud.inetutils.preferred-networks=10.244

# 忽略eth0, 支持正则表达式
spring.cloud.inetutils.ignored-interfaces=eth0

参考地址：https://blog.csdn.net/qq_29824445/article/details/108622838

  1. 查看版本对应，客户端是2.3.0的，服务端也是2.3.0的。未发现异常。

  1. 尝试远程调试xxl-job的代码，同样无效

在Dockerfile中修改jvm启动命令如下：


FROM 192.168.193.105/test-cloud/base_backend:0.0.5

ENV PARAMS=""

COPY ./xxl-job-admin-2.3.1-SNAPSHOT.jar /app

EXPOSE 5005

CMD ["sh", "-c", "java -jar $JAVA_OPTS -agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=5005 xxl-job-admin-2.3.1-SNAPSHOT.jar $PARAMS"]

添加-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=5005到Dockerfile中，把jar包拷贝到同目录，重新打镜像：


$ docker build -t 192.168.193.105/test-cloud/xxl-job-admin-test:0.0.2 .

  1. 将xxl-job的执行端口以nodePort形式暴露出来，重新进行调用测试

修改客户端服务部署的内容，修改les后端服务的k8s服务部署文件，如下：


$ vim test-les-app.yaml
// 添加以下内容
---
apiVersion: v1
kind: Service
metadata:
  name: test-les-xxl-job
  namespace: test-les
  labels:
    app: test-les-app
spec:
  type: NodePort
  ports:
    - port: 9988
      name: xxl-job
      targetPort: 9988
      nodePort: 30988
  selector:
    app: test-les-app
---
...

在xxl-job-admin中，修改执行器地址，由


http://test-les-app.test-les.svc.cluster.local:9988

变更为：

http://192.168.166.111:30988,http://192.168.166.112:30988,http://192.168.166.113:30988,http://192.168.166.114:30988

变更后保存，开启定时任务执行。

经过测试发现，除了服务重启更新之外，服务会存在定时任务执行失败的情况，其余情况下会正常执行。执行失败的情况也不再是ip地址有变更的情况了，ip地址也不发生变更了。

总结

善用k8s自己的特性，来辅助解决问题，实现业务逻辑！
