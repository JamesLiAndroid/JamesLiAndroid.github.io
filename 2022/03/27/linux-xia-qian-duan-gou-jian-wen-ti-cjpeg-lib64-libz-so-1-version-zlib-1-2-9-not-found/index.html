前端的构建命令为：


yarn install --verbose

yarn run build

根据同前端同志的交流，在windows下使用yarn编译没有问题。实际上问题的核心就在于*/lib64/libz.so.1: version `ZLIB_1.2.9’ not found *这条日志上。

解决方式

下载zlib 1.2.9版本的源码，编译安装后，重新设置libz.so.1文件指向即可，过程如下：


$ wget http://www.zlib.net/fossils/zlib-1.2.9.tar.gz

$ tar -zxvf zlib-1.2.9.tar.gz

$ cd zlib-1.2.9

$ ./configure --prefix=/lib64

$ make

$ sudo make install

// 创建软链接
$ sudo ln -snf /lib64/libz.so.1 /lib64/lib/libz.so.1.2.9

上述设置完成后，重新构建前端服务，即可成功。

补充：zlib库要慎重修改

对于客户的服务器，我们没有完整的root权限，只有分配给我们的用户有sudo执行的权利。在第一次尝试修改libz.so.1，想要把新编译好的libz.so.1.2.9替换为libz.so.1，操作如下：


// 备份原来的包，先用mv命令
$ cd /lib64 && sudo mv libz.so.1 libz.so.1.bak

$ cd ~/zlib-1.2.9 && sudo cp libz.so.1.2.9 /lib64/libz.so.1
sudo: error in /etc/sudo.conf, line 19 while loading plugin "sudoers_policy"
sudo: unable to load /usr/libexec/sudo/sudoers.so: libz.so.1: cannot open shared object file: No such file or directory
sudo: fatal error, unable to load plugins

这样就导致了sudo命令不可用，在没有root权限的情况下，属于灭顶之灾了。所以尽可能不要修改这个文件，如果修改也务必谨慎操作！尽可能使用cp命令进行备份后，再操作软连接设置，这样不至于出现sudo用不了的情况！
