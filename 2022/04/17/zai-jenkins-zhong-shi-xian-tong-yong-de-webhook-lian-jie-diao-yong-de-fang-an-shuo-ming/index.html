在jenkins中实现通用的webhook连接调用的方案说明

前言

由于公司层面更换了办公通讯软件，由企业微信更换为飞书，导致我们之前在jenkins中配置的企业微信webhook构建信息发布，无法在飞书中使用。由于我司DevOps人员在编码层面水平实在不咋地，没法从修改源码的角度去重新做插件来实现。所以需要找到一个更快速、更便捷而且以他们能看懂的方式去实现构建任务状态的webhook告警。

插件使用

选用Post Build Task插件来实现，该插件允许用户依据构建日志的输出，在构建任务执行完成、执行中断、执行失败时，执行一个shell/批处理任务来进行构建之外的一些操作，例如清理文件夹、发送请求等这样的操作。在后续的操作之前，需要先安装这个插件到jenkins中！

输入全局变量信息

以后端项目为例子进行构建，在Build模块中（如果有Pre Build模块更好，可以放在这里面），在构建任务正式开始前，创建Execute Shell脚本执行框，如下：


#!/bin/bash
echo 'send_to_lark_notification'

jenkins配置用户登录密钥

以管理员权限登录jenkins，直接访问http://${JENKINS_URL_PORT}/user/admin/configure这个地址，找到api token的位置，点击Add New Token，填入一个名字，会给你生成一个token信息，该token信息只会出现一次，需要自行复制后保存。操作如下：

这时候再请求时jenkins的api调用接口获取信息时，不再需要输入密码，使用api token就可以了。

以飞书webhook为例进行脚本编写

在构建任务最后Post-build Actions模块中，创建Post build task执行框，填写log text信息，同上面shell中输出的文字相同，如下：

然后编写Script脚本信息如下：


#!/bin/bash
# 单个构建任务的完整URL信息
JOB_URL="${JENKINS_URL}job/${JOB_NAME}"
echo $JOB_URL
# WEBHOOK链接信息
WEBHOOK_URL=''
# jenkins登录信息
jenkins_uname=''
# jenkins登录密钥信息
jenkins_api_token=''
getBuildState(){
  buildNr=$1
  curl -v -L ${JOB_URL}/${buildNr}/api/json --user $jenkins_uname:$jenkins_api_token | grep -Po '"result":\s*"\K\w+'
}
state=$(getBuildState ${BUILD_NUMBER}  )
string1=$BUILD_DISPLAY_NAME
string2=$JOB_BASE_NAME
nowTime=$(date "+%Y-%m-%d %H:%M:%S")
# 如果有错误信息生成错误信息链接
errUrl=${JOB_URL}/${BUILD_NUMBER}/console
echo $state
echo $errUrl
if [[ "x${state}" == "xSUCCESS" ]] ; then
   curl -X POST -H "Content-Type: application/json" \
        -d '{"msg_type":"post","content": {"post": {"zh_cn": {"title": "发布结果通知","content": [[{"tag": "text","text": "'"应用：后端构建通知专用\n系统：$string2\n构建：$string1\n状态：成功\n日期：$nowTime"'"}]]} } }}' \
$WEBHOOK_URL
else
   curl -X POST -H "Content-Type: application/json" \
        -d '{"msg_type":"text","content":{"text":"'"应用：后端构建通知专用\n系统：$string2\n构建：$string1\n状态：<br>失败</br>\n日期：$nowTime\n错误信息地址：$errUrl"'"}}' \
$WEBHOOK_URL
fi

脚本中的webhook地址、jenkins用户和api token需要自行获取并输入，输入完成之后，可以使用该脚本进行测试。
